# 餐厅系统 — 使用手册

面向店主和员工的通俗指南，无需任何技术背景。

> 📷 _带此标记的是截图占位符。请在对应位置替换为您平板/电脑上的真实截图。_

本应用分为两部分：

- **商家端（收银 POS）** — 在 Windows 10/11 电脑或安卓平板上运行。本手册主要介绍它。
- **顾客端** — 顾客用来预订自取，见 [§9](#9-顾客端)。

所有功能均可**离线**使用——没有网络时收银照常运行。云功能（在线订单、备份）是可选的，运行在您自己的账户上；我们不托管任何数据，也**没有强制订阅**。

## 1. 界面导航

打开商家端后，侧边/底部有以下标签页：

| 标签 | 用途 |
|---|---|
| **订单** | 开堂食/外带单、发送到厨房、收款 |
| **菜单** | 分类、菜品、价格、选项、照片 |
| **收件箱** | 顾客的在线预订单（需云设置） |
| **报表** | 每日销售汇总与历史订单 |
| **设置** | 税务、打印机、结账选项、云、语言 |
| **管理** | 员工账号与角色 |

> 📷 _[截图：含标签页的主界面]_

**语言：** 随时在 **设置 → 语言** 切换（English、中文，或跟随系统）。小票统一以英文打印。

## 2. 首次设置

打开**设置**，从上到下逐项配置。只需设置一次。

### 税务
- **销售税率** — 您当地的税率（如 13）。仅适用于新订单；之后修改**不会**改写已有订单。

### 结账
- **服务费** — 对每笔订单自动收取的费用（设为 0% 表示不收）。
- **折扣预设** — 快捷折扣按钮，如 `5、10、15`。
- **无需经理的折扣上限** — 不超过此额度的折扣任何员工都可直接打；更大的折扣需要经理 PIN。

### 打印
- **网络打印机** — 收据打印机的 IP 地址和端口（通常为 **9100**）。按打印机选择 **58 毫米** 或 **80 毫米** 纸。
  - 点 **搜索打印机** 自动扫描网络，或手动输入 IP。
  - **测试打印** 发送一张测试页以确认连接正常。
- **小票上的商家名称** 和 **小票页脚** — 顾客小票顶部和底部打印的内容。
- **打印队列** — 显示待打印/失败的任务，可重试或丢弃。

> 📷 _[截图：设置 → 打印]_

### 餐桌（堂食用）
- 添加您的餐桌（如 `1、2、Patio A`）。堂食开单时会询问餐桌。

### 在线点餐（可选）
- **最短取餐时间** — 顾客最早可要求的取餐时间（随菜单一起发送）。
- **新订单提示音** — 有新在线订单时播放提示音。

### 第二名称显示
若菜品带有第二名称（如英文名下方的中文名），可选择它显示在何处：
- **在点单界面**、**在厨房单**、**在顾客小票** — 每项可单独开关。
- **第二名称语言** — 顾客端使用此语言时，会优先看到第二名称。

### 支付
- **刷卡终端：手动输入** — 目前由员工在独立刷卡机上输入金额并记录结果。（集成式 Moneris Go 支持将于稍后推出。）

## 3. 员工与角色

可选，但当不止一人操作收银时建议启用。

1. 进入 **管理 → 创建第一个店主**。在此之前，所有人都拥有完整权限。
2. 在 **员工与角色** 中添加员工。每人设置 **姓名**、**角色** 和 **4 位 PIN**。

**角色：**
- **店主** — 完整权限，包括管理员工。
- **经理** — 可批准大额折扣及日常管理。
- **服务员** — 开单与收款。

员工用 **姓名 + PIN** 登录（因此两人即便 PIN 相同也不会混淆）。用 **切换用户** 把平板交给他人，或 **退出登录**。

> 📷 _[截图：管理 → 员工与角色]_

## 4. 开单点餐

1. 在 **订单** 标签，开始一笔 **堂食**（选餐桌）或 **外带** 订单。
2. 点一个**分类**，再点**菜品**，即可加入右侧订单。
   - 若菜品有选项（如*规格*、*加料*），会弹出选择框——选好后点 **加入订单**。
3. 按需调整：
   - **＋ / －** 增减数量。
   - 数量为 1 时的 **－** 会**作废**该项（保留在历史中，不会被悄悄删除）。
4. **折扣**（可选）：点 **添加折扣**，选预设或输入百分比。超过"无需经理"上限的折扣会要求经理 PIN。
5. **发送到厨房** 打印厨房单。之后仍可继续修改，并可 **重新打印厨房单**。

合计面板按 **小计 → 折扣 → 服务费 → 税 → 合计** 显示，让您随时看清价格构成。

> 📷 _[截图：含菜品与合计的开单界面]_

## 5. 收款

点 **收款 {金额}**，弹出收款窗口并显示应付金额。

### 现金（含找零）
1. **金额** 已自动填为应付余额。
2. （可选）在 **实收现金** 中输入顾客给的金额——窗口会自动显示 **应找零**。
3. 点 **现金**，完成。

> 📷 _[截图：显示应找零的收款窗口]_

### 刷卡
1. 点 **刷卡**。
2. 在独立刷卡机上输入该金额。
3. 记录刷卡机显示的结果：**已批准** 或 **已拒绝**。被拒会被记录但不计为已付。

### 拆分账单

根据顾客的拆分方式，有**两种**做法：

**A. 按金额拆分** — 把 **金额** 调低为账单的一部分并收款；对每个人重复。订单在余额清零前保持未结。

**B. 按菜品拆分** *（订单含 2 项及以上时）* — 点 **按菜品拆分**：
1. 勾选某人点的菜品。
2. 按钮显示 **为所选收款 — {金额}**。该金额已包含这组菜品应分摊的税、服务费和折扣。
3. 点击后照常用 **现金** 或 **刷卡** 收款。
4. 这些菜品会被标为 **已付** 并从列表移除。对下一个人重复，直到全部付清。

> 📷 _[截图：勾选了部分菜品的"按菜品拆分"]_

当**全部余额**付清后，订单关闭并自动打印**顾客小票**。部分付款则保持订单未结。

## 6. 菜单管理

在 **菜单** 标签：

- **分类** 用来归类菜品（如*前菜*、*寿司卷*）。先建分类，再加菜品。
- **新建菜品** — 设置**名称**和**价格**。可选项：
  - **菜品编号**（如 `A01`）、**第二名称**（如本地语言名称）、**描述**。
  - **自定义字段**（配料、过敏原、辣度、备注等）。
  - **照片**（先保存菜品，再添加照片）。
  - **选项组** — 可复用的选项集合，如*规格*或*加料*，可设最少/最多选择数和加价（可为负，如 `-0.50`）。
- **加载示例菜单** — 添加 29 个菜品的中英双语示范菜单（忆寿司），便于立即试用点单与打印。
- 用删除操作删除菜品或分类（历史订单各自保留快照，因此不受影响）。

> 📷 _[截图：含分类与菜品的菜单标签]_

### 从照片导入菜单
在 Windows 或安卓上可用 **从照片导入** 加速录入：
1. 先建一个**模板**——在示例照片上拖一个框框住一个菜品，并标出**编号 / 名称 / 价格**的位置。
2. **选择菜单照片**，选好模板和分类，对每道菜点 **采集菜品**；应用会自动识别文字。
3. **审核草稿** 后 **全部保存**。

> 注：照片导入需要 OCR 语言包。Windows 上：**设置 → 时间和语言 → 语言**（添加中文或英文）。Linux 上不可用。

## 7. 在线订单（收件箱）

让顾客预订自取。需要可选的云设置（[§10](#10-云备份与隐私)）并已发布菜单。

1. 在 **收件箱** 点 **发布菜单**，把当前菜单发送到您的店铺。菜单变动后请重新发布。
2. 顾客扫描您的 **顾客连接码**（在设置中）添加餐厅并下预订单。

订单进来后：

- **新预订单** — 每单可 **接受**、**拒绝** 或 **建议时间**（建议不同的取餐时间，顾客随后确认或拒绝）。
- **准备中** — 已接受的订单。做好后点 **标记为就绪**，系统会通知顾客。
- **可取餐** — 顾客来取时，点 **标记为已取餐** 完成订单。您或顾客任一方都可操作，且两端同步更新。

> 📷 _[截图：含 新 / 准备中 / 可取餐 区块的收件箱]_

## 8. 报表

**报表** 标签每次显示一天（用 **前一天 / 后一天** 切换）：

- **已收** — 收款总额，按收款和小费细分。
- **菜品销量** — 各菜品售出数量。
- **历史订单** — 当天所有已结订单（已作废的会标注）；点开可 **重新打印小票**。
- **销售总额**、**税**、**小费**、**收款合计** 汇总。

> 📷 _[截图：某一天的报表]_

## 9. 顾客端

顾客的操作（供您协助他们时参考）：

1. **添加餐厅** — 扫描您的 **顾客连接码**、上传它的照片，或手动输入链接。无需账号或登录。
2. **浏览菜单** — 点菜品查看描述、选项和照片，选择**数量**后加入购物车。
3. **结账** — 填写姓名（电话可选），选**取餐时间**（不早于您设定的最短时间），然后 **提交预订单**。
4. **追踪** — 状态从 *等待中 → 准备中 → 可取餐* 实时更新。若您建议了其他时间，顾客可 **确认** 或 **拒绝**。取餐后由顾客（或您）标记为 **已取餐**。
5. **我的信息** — 顾客可保存姓名/电话/邮箱及通知偏好，方便快速结账。

> 📷 _[截图：顾客端菜单与结账]_

## 10. 云备份与隐私

POS 采用**离线优先**——平板自身存储是数据的唯一真实来源，没有网络也能完整运行。

云功能（在线订单 + 备份）是**可选的**，运行在**您自己的免费 Supabase 项目**上——我们不托管任何数据：

- **设置 → 云同步**：用您创建的餐厅账号登录，输入 **项目 URL** 和 **anon key**，然后 **立即同步**。
- **从云端恢复** 可把完整历史拉取到新的或重置过的平板上（已有数据会合并，绝不清除）。
- 您的私密数据始终保密：使用点餐应用的顾客只能读取您已发布的菜单和他们自己的订单——看不到您的销售、员工或其他顾客的订单。

> 设置是一次性的技术步骤，详见 [CLOUD_SECURITY.md](CLOUD_SECURITY.md)。若不使用云功能，可忽略本节。

## 11. 疑难排查

| 问题 | 解决方法 |
|---|---|
| **找不到打印机 / 不出单** | 设置 → 打印 → **搜索打印机**，或检查打印机 IP、是否在同一网络、端口是否为 **9100**。用 **测试打印** 确认。 |
| **在线订单报授权错误（400）** | 在 设置 → 云同步 中**退出再重新登录**以刷新登录凭证。 |
| **登录提示 "no host specified in URL"** | 您的项目 URL 开头缺少 `https://`，请补上。 |
| **"未找到 OCR 语言包"（照片导入）** | Windows → 设置 → 时间和语言 → 语言——添加中文或英文。 |
| **应用无法在店里的电脑上运行** | 需要 **Windows 10 或 11**（不支持 Windows 7/8）。也可改用安卓平板。 |
| **顾客的"已取餐"/"确认时间"按钮报错** | 餐厅的云项目需要应用安全规则——一次性设置步骤（见 CLOUD_SECURITY.md）。 |
